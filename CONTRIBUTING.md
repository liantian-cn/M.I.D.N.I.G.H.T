# Contributing

## 维护模式说明

本项目当前以**单维护者模式**运行。欢迎提交建议与修复，但合并节奏与方向由维护者统一控制。

## 贡献入口

1. 先通过 Issue 描述问题或提议。
2. 达成方向一致后，再提交 PR。
3. PR 应聚焦单一主题，避免混合大量无关改动。

## 基本要求

1. 文档优先：涉及协议行为变化时，先更新 `docs/` 下相关规范。
2. 变更可追踪：协议相关调整需更新 `CHANGELOG.md`。
3. 风险边界：不得提交破坏公平性的自动化/外挂逻辑。
4. 保持最小改动面，便于审阅和回滚。

## 提交建议

推荐提交前缀：

- `docs:` 文档更新
- `chore:` 脚手架与维护项
- `feat:` 新能力
- `fix:` 缺陷修复
