# FAQ

## 1. 为什么使用像素矩阵通信？

为了将复杂计算迁移到游戏外执行：游戏内负责“画数据”，游戏外负责“读数据”。

## 2. 为什么协议要文档先行？

DejaVu 与 Terminal 是两端独立实现。协议未对齐会直接导致解析失败，因此先固定文档契约。

## 3. 抗锯齿和颜色偏差如何处理？

解析侧应采用颜色容差匹配，而不是绝对 RGB 硬匹配；并结合帧完整性标记做容错。

## 4. 为什么当前版本对齐游戏版本？

项目版本与 WoW 客户端版本保持一致，当前基线为 `12.0.1.66198`，便于两端协议与运行环境对齐。

## 5. 这个项目是开箱即用工具吗？

不是。该项目用于技术方案与协议实践，不提供可直接投入使用的成品。

## 6. 如何判断文档是否需要同步更新？

凡是涉及：

1. Cell/MegaCell/BadgeCell 语义变化
2. Matrix 布局变化
3. 协议字段变化

都应同步更新 `docs/` 与 `CHANGELOG.md`。
