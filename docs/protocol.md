# 通信协议（草案）

- 文档版本：`12.0.1.66198`
- 状态：`draft`（草案）
- 适用范围：DejaVu（绘制端）与 Terminal（解析端）之间的像素通信契约

## 1. 设计目标

1. 将游戏内状态编码为可截图读取的像素矩阵。
2. 将复杂计算迁移到游戏外进行，降低游戏内计算压力。
3. 保持绘制端与解析端的松耦合与可独立演进。

## 2. 角色模型

1. **Producer（DejaVu）**：将状态数据编码并绘制到 Matrix。
2. **Consumer（Terminal）**：采样像素并按协议恢复结构化数据。

## 3. 草案常量（可变）

| 常量               | 当前值         | 状态  | 说明     |
| ---------------- | ----------- | --- | ------ |
| `CELL_SIZE`      | `4x4`       | 草案  | 基础像素单元 |
| `MEGA_CELL_SIZE` | `8x8`       | 草案  | 图标单元   |
| `MATRIX_ANCHOR`  | `TOP_RIGHT` | 草案  | 矩阵锚点   |

> 以上常量在 `draft` 阶段不保证跨版本兼容。

## 4. 数据帧概念模型

本阶段只定义语义，不锁定位宽或具体编码位序：

1. **Header（帧头）**
   - `protocol_version`
   - `frame_type`
   - `sequence_id`
   - `timestamp_hint`
2. **Payload（载荷）**
   - 由多个逻辑字段组成，映射到若干 Cell/MegaCell/BadgeCell。
3. **Integrity（完整性）**
   - 校验字段（例如 checksum/marker），用于识别截帧与脏帧。

## 5. 传输生命周期（概念）

1. Producer 刷新 Matrix 帧。
2. Consumer 截图并定位 Matrix 边界。
3. Consumer 进行 Cell 级采样与颜色匹配。
4. Consumer 组合字段并执行完整性校验。
5. 校验失败时丢帧并等待下一帧。

## 6. 版本协商策略（概念）

1. Consumer 必须读取 `protocol_version`。
2. 若版本不兼容，Consumer 进入降级模式：
   - 不执行高风险推断。
   - 输出版本不匹配状态给上层。
3. 新增字段应优先采用“可忽略扩展”策略，降低破坏性升级概率。

## 7. 错误与容错（概念）

1. 抗锯齿、缩放与截图压缩可能导致颜色偏移。
2. Consumer 应使用容差匹配而非绝对 RGB 硬匹配。
3. 连续失败达到阈值后，应触发重定位而不是持续错误解码。

## 8. 非目标

1. 本文档不定义自动化操作策略。
2. 本文档不提供绕过反作弊的实现细节。
3. 本文档不承诺 `draft` 阶段的长期兼容性。
