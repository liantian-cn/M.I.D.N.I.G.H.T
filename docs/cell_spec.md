# Cell 结构规范（Draft）

- 文档版本：`12.0.1.66198`
- 状态：`draft`

## 1. 单元类型

1. **Cell**
   - 尺寸：`4x4`
   - 用途：承载基础状态位与小型字段编码。
2. **MegaCell**
   - 尺寸：`8x8`
   - 用途：承载图标级语义或较高信息密度字段。
3. **BadgeCell**
   - 基于 MegaCell 扩展
   - 用途：在角标区域附加状态类别（如魔法/中毒等）。

## 2. 颜色与语义

1. 颜色定义以 `color_palette.md` 为唯一来源。
2. 消费端应使用容差比较，避免因抗锯齿导致误判。

## 3. Badge 角标位置（Draft）

建议角标象限语义如下（后续可调整）：

1. 左上：控制/限制类标记
2. 右上：可交互提示类标记
3. 左下：持续性效果类标记
4. 右下：优先级或置信度类标记

> 本阶段仅定义结构方向，不冻结具体编码表。

## 4. 渲染一致性要求

1. 同一字段必须在所有帧保持同一位置与同一颜色语义。
2. Cell 之间应保留最小间隔，降低颜色污染。
3. 不同单元类型不得在同一逻辑槽位混用。

## 5. 解析容错建议

1. 当单元颜色匹配冲突时，优先使用帧级完整性信号进行回退。
2. 当角标读取失败时，允许输出主语义并标记角标未知状态。
