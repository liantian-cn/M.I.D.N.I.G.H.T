# Matrix 布局规范（Draft）

- 文档版本：`12.0.1.66198`
- 状态：`draft`

## 1. 布局目标

1. 矩阵必须固定在屏幕可预测区域，便于外部程序快速定位。
2. 布局需尽量降低 UI 元素遮挡与动态抖动影响。

## 2. 锚点约定（Draft）

1. 默认锚点：`TOP_RIGHT`（屏幕右上角）。
2. 使用安全边距避免贴边采样误差。
3. 允许未来引入可配置锚点，但默认值不变。

## 3. 坐标系

1. 原点采用屏幕左上角坐标系（常见 2D UI 坐标）。
2. Matrix 位置由锚点偏移定义：
   - `offset_x`：向左为正偏移（相对右上角锚点）。
   - `offset_y`：向下为正偏移（相对右上角锚点）。

## 4. 网格组织（概念）

1. Matrix 由规则网格组成。
2. 网格单元可以是：
   - `Cell`（4x4）
   - `MegaCell`（8x8）
   - `BadgeCell`（带角标语义）
3. 网格尺寸在 `draft` 阶段允许调整，但应在变更日志记录。

## 5. 显示与采样建议

1. 建议禁用影响像素准确性的缩放/滤镜路径（若环境允许）。
2. Consumer 应支持矩阵局部重定位，处理偶发 UI 偏移。
3. 采样前应先完成矩阵边框或标记识别，以降低误检。

## 6. 兼容性注意

1. 不同分辨率和 UI 缩放下，像素边界可能出现半像素映射问题。
2. 建议在协议中保留同步标记区域，用于校准当前帧。
