# DejaVu 个人注释与结构风格

这份文档记录当前个人代码风格要求，适用于 DejaVu 侧显示模块、事件模块和配置回调模块。

## Cell / Bar 实例注释

- 所有 `Cell`、`CharCell`、`BadgeCell`、`Bar`、`MegaCell` 实例创建前都写三行注释。
- 固定格式：

```lua
-- x:45 y:14
-- 用途：单位施法图标
-- 更新函数：updateCastAndChannel
```

- 坐标允许直接写源码里的表达式，例如 `BASE_X - 20`、`x + 1`、`y - 1 + i`，不要求强行化简。
- 如果一个实例会被多个刷新函数直接维护，第三行按实际函数名并列写全。
- 如果实例是注释掉的旧代码，也补相同格式注释，但保持注释状态。

## update 函数注释

- 所有负责把状态写回显示单元的刷新函数，都写三行注释。
- 固定格式：

```lua
-- 说明：更新玩家的施法、通道和蓄力状态。
-- 依赖事件更新：UNIT_SPELLCAST_* 事件。
-- 依赖定时刷新：2 秒。
```

- 没有事件驱动时，第二行写 `无`。
- 没有定时刷新时，第三行写 `无`。
- 首次刷新专用函数、回调型更新函数、配置映射函数，也按同一格式写。

## event 注册注释

- 所有活动事件注册前都写三行注释。
- 固定格式：

```lua
-- UNIT_SPELLCAST_INTERRUPTED
-- 事件说明：施法被打断时刷新施法、通道和蓄力状态。
-- 对应函数：updateCastAndChannel
```

- 事件函数要紧跟在对应注册语句后面。
- 一个事件里如果直接调用多个刷新函数，第三行把实际调用的函数名写全。
- 注释掉的旧事件代码也补同样格式，但保持注释。

## 事件模块顺序

- 标准事件模块顺序固定为：
  1. `eventFrame` 构建
  2. cell / bar 实例构建
  3. update 函数构建
  4. event 注册和紧邻的事件函数
  5. 路由，先定时再事件
  6. 首次刷新
- 如果文件只有 `OnUpdate` 没有活动事件，不强行补 `OnEvent` 路由，但仍保留“路由 -> 首次刷新”的尾部顺序。
- 首次刷新块统一放在所有路由之后。

## 回调型文件例外

- `Config.lua`、黑名单显示、配置映射这类回调驱动文件，不硬套 `eventFrame` 骨架。
- 这类文件按下面顺序整理：
  1. 实例构建
  2. 刷新函数 / 回调函数
  3. 回调注册
  4. 首次刷新

## 旧代码处理

- 注释掉的旧事件、旧 update、旧实例代码，当前策略是“补注释，但不启用”。
- 不因为整理注释而顺手恢复旧逻辑。
